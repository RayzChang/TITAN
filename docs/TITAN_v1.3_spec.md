## 一、策略用途

本策略用於幣安 USDT-M 永續合約自動交易。
只交易兩個標的：

- BTC/USDT
- ETH/USDT

策略邏輯分三層：

- **日線**：找箱體、判斷大結構、加倉方向確認
- **1 小時**：進場訊號、第二段出場
- **4 小時**：第一段出場、加倉價格確認

## 二、固定參數

| 項目 | 設定值 |
|------|--------|
| 總本金 | 5,000 USDT |
| 每次開倉保證金 | 100 USDT |
| 槓桿 | 100 倍 |
| 保證金模式 | 全倉（Cross） |
| 每次開倉名義倉位 | 10,000 USDT |
| 最大同時持有標的數 | 2（BTC + ETH 各一） |
| 每個標的同時只允許一個總部位 | 是（加倉算同一部位） |
| 系統掃描間隔 | 每 30 秒一次 |

## 三、統一判定原則

### 3.1 K 棒規則
- 所有策略判斷一律只使用**已收完的 K 棒**
- K 棒時間一律以**幣安合約 UTC 收線**為準
- 如果沒有新的已收線，系統只更新價格與持倉資訊，不產生新訊號

### 3.2 MACD 參數
- 1 小時、4 小時、日線的 MACD 參數全部固定為 **12 / 26 / 9**

### 3.3 判定基準
- 箱體突破、跌破、MACD 交叉、箱體失效，全部以**收盤價**判定
- 停損單觸發價使用**標記價格（Mark Price）**
- 進場、加倉、平倉一律使用**市價單**

### 3.4 成交量
- 日線平均成交量固定用**最近 20 根已收日線**計算

### 3.5 回測限制聲明
- 回測無標記價格數據，停損以收盤價近似，結果與實盤會有偏差
- 此為已知限制，不影響策略邏輯正確性

## 四、箱體建立規則

### 4.1 觸發時機

| 情境 | 處理方式 |
|------|---------|
| 全新部署（無 state.json） | 讀取 settings.yaml 的 manual_boxes 當作啟動種子 |
| 正常重開（有 state.json） | 從 state.json 恢復箱體狀態，**不重讀 manual_boxes** |
| 舊箱體失效 + 當下無持倉 | 自動建立新箱體 |
| 舊箱體失效 + 當下有持倉 | 標記失效、禁開新倉，等平倉後再自動重建 |

**重點：manual_boxes 是「冷啟動種子」，不是「永久覆寫」。只在全新部署時讀取一次。**

### 4.2 自動建箱流程

1. 取最近 **120 根**已收日線
2. 找出成交量最大的一根，當作箱體錨點
   - 錨點候選條件：後面至少還有 **4 根**已收日線（否則跳過，找次大量的候選）
3. 從錨點開始，往後固定取 **5 根**已收日線當作形成期（含錨點本身 + 後 4 根）
4. 固定下緣 = 形成期內所有 K 棒的**最低價**
5. 第一層上緣 = 形成期內所有 K 棒的**最高價**
6. 上緣清單初始值 = [第一層上緣]
7. 每次重建箱體時，舊的上緣清單全部清空，重新開始

### 4.3 為什麼下緣用形成期最低而非單根錨點最低

以 ETH 為例：錨點日（2025/2/5）最低價 = 1813，但隔天（2/6）最低 = 1736。
朋友實際畫的箱體下緣 = 1740，與形成期最低價（1736）僅差 4 點。
若只用錨點那根的最低價，箱體下緣會偏高，容易被假跌破觸發。

## 五、上緣延伸規則

系統只追蹤「目前最高那一層上緣」。

1. 如果任一根**已收日線**的收盤價**站上**目前最高上緣 → 系統進入「上破追蹤」狀態
2. 進入上破追蹤後，持續記錄這一段行情中的**最高日線高點**
3. 只要日線收盤仍然**站在**舊最高上緣之上，就持續更新最高價
4. 如果之後任一根已收日線收盤**重新跌回**舊最高上緣下方 → 視為一次**假突破**
5. 這次假突破期間記錄到的最高日線高點，加入上緣清單，成為新的最高上緣
6. 舊上緣全部保留，不刪除

**白話：上緣只在「確認是假突破」之後才會新增一層，不會在突破過程中就急著加。**

## 六、箱體失效規則

箱體失效必須**同時**滿足兩個條件：

1. 已收日線收盤價 **< 固定下緣**（嚴格小於）
2. 該根日線成交量 **>= 最近 20 根已收日線平均成交量 × 1.5**

### 失效後的處理（含持倉保護）

| 當下狀態 | 處理方式 |
|---------|---------|
| 無持倉 | 立刻自動重建新箱體 |
| 有持倉 | 標記箱體失效 → 禁止開新倉 → 現有部位照原止損/止盈規則繼續運作 → 全部平倉後再重建 |

---

## 七、做多進場規則

做多訊號必須在**同一根已收 1 小時 K 棒**上同時成立：

1. 收盤價 **> 任一層上緣**（嚴格大於）
2. 1 小時 MACD 柱狀圖**由負轉正**（當根 > 0 且 前根 <= 0）

### 補充規則

- 如果同一根 K 棒同時突破多個上緣，以「被突破的最高那一層上緣」當作**本次交易的基準上緣**
- 同一標的若已持有部位，**不得**重複開新倉
- 箱體已標記失效時，**禁止**開新倉

### 防重複追單

如果價格一直停留在上緣上方，不因後面再次出現金叉就重複進場。

重置條件：必須先有一根 1 小時收盤 **< 本次進場的基準上緣**（嚴格小於），之後再重新突破 + 金叉，才算新的做多訊號。

---

## 八、做空進場規則

做空訊號必須在**同一根已收 1 小時 K 棒**上同時成立：

1. 收盤價 **< 固定下緣**（嚴格小於）
2. 1 小時 MACD 柱狀圖**由正轉負**（當根 < 0 且 前根 >= 0）

### 補充規則

- 同一標的若已持有部位，**不得**重複開新倉
- 箱體已標記失效時，**禁止**開新倉

### 防重複追單

如果價格一直停留在下緣下方，不因後面再次出現死叉就重複進場。

重置條件：必須先有一根 1 小時收盤 **> 固定下緣**（嚴格大於），之後再重新跌破 + 死叉，才算新的做空訊號。

---

## 九、開倉後的固定止損

### 做多止損
- 止損價 = 固定下緣 × 0.99

### 做空止損
- 止損價 = **進場當下**的最高上緣 × 1.01
- **做空止損凍結規則**：空單進場後，就算之後箱體新增了更高的上緣，這筆空單的止損價也**不改**

### 通用規則
- 開倉成功後，立刻掛出對應的停損單
- 在第一段出場之前，停損價不移動
- 如果有加倉，只更新停損單**數量**，不改停損**價格**

---

## 十、第一段出場規則（4H 觸發）

### 多單第一段出場
- 持有多單時
- 任一根已收 4 小時 K 棒的 MACD 柱狀圖**由正轉負**（當根 < 0 且 前根 >= 0）
- 立刻市價平掉目前總部位的 **50%**

### 空單第一段出場
- 持有空單時
- 任一根已收 4 小時 K 棒的 MACD 柱狀圖**由負轉正**（當根 > 0 且 前根 <= 0）
- 立刻市價平掉目前總部位的 **50%**

### 第一段出場完成後
- 剩餘部位的停損移到**整體平均開倉價**（保本線）
- 此後部位進入「保本保留」狀態

### 同時觸發規則
- 如果 TP1 觸發的那一刻，最新一根已收 1 小時 K 棒**也已經**出現第二段出場的反向信號 → 剩餘部位**直接全部平倉**，不留半倉

---

## 十一、第二段出場規則（1H 觸發）

只有在第一段出場完成後，第二段出場才會生效。

### 多單第二段出場
- 任一根已收 1 小時 K 棒的 MACD 柱狀圖**由正轉負**
- 剩餘部位全部市價平倉

### 空單第二段出場
- 任一根已收 1 小時 K 棒的 MACD 柱狀圖**由負轉正**
- 剩餘部位全部市價平倉

---

## 十二、加倉規則

每一筆交易**最多只允許加倉一次**。
加倉只允許發生在「**第一段出場之前**」。

### 加倉額度
- 再加 100 USDT 保證金
- 槓桿維持 100 倍

### 做多加倉條件（必須同時滿足）
1. 連續 **3 根**已收 4 小時 K 棒的最低價都 **> 本次交易的基準上緣**（嚴格大於）
2. 最新一根已收日線的 MACD 柱狀圖 **> 0**（多方持續狀態）

### 做空加倉條件（必須同時滿足）
1. 連續 **3 根**已收 4 小時 K 棒的最高價都 **< 固定下緣**（嚴格小於）
2. 最新一根已收日線的 MACD 柱狀圖 **< 0**（空方持續狀態）

### MACD 使用方式統一說明
| 用途 | 看什麼 | 白話 |
|------|--------|------|
| 進場（1H） | 柱狀圖**翻轉事件**（剛由負轉正 / 由正轉負） | 看剛發動 |
| TP1 出場（4H） | 柱狀圖**翻轉事件** | 看剛反轉 |
| TP2 出場（1H） | 柱狀圖**翻轉事件** | 看剛反轉 |
| 加倉確認（日線） | 柱狀圖**持續狀態**（仍在 0 軸上方或下方） | 看趨勢還活著 |

### 加倉完成後
- 整體平均開倉價重新計算
- 第一段與第二段出場都以加倉後的**總部位**為準
- 原本止損價**不變**，只更新停損單**數量**

---

## 十三、持倉限制

- BTC 同一時間最多只有**一個**總部位
- ETH 同一時間最多只有**一個**總部位
- 同一標的**不可**同時做多又做空
- 全系統同時最多持有**兩個**標的
- 加倉不算新標的持倉
- 已進入第一段出場後，**不再允許**加倉

---

## 十四、策略狀態流程

### 14.1 狀態定義

每個標的只允許在以下狀態中切換：

| 狀態代碼 | 名稱 | 說明 |
|---------|------|------|
| IDLE | 空倉 | 無部位，等待訊號 |
| LONG | 持有多單 | 可加倉 |
| LONG_HALF | 多單已出一半 | 保本保留中，不可加倉 |
| SHORT | 持有空單 | 可加倉 |
| SHORT_HALF | 空單已出一半 | 保本保留中，不可加倉 |
| BOX_INVALID | 箱體失效等待重建 | 有持倉未平完，禁開新倉 |

### 14.2 狀態切換表

| 當前狀態 | 事件 | 新狀態 |
|---------|------|--------|
| IDLE | 做多訊號成立 | LONG |
| IDLE | 做空訊號成立 | SHORT |
| LONG | 4H MACD 反轉 | LONG_HALF |
| SHORT | 4H MACD 反轉 | SHORT_HALF |
| LONG / SHORT | 停損觸發 | IDLE |
| LONG_HALF / SHORT_HALF | 1H MACD 反轉 或 停損觸發 | IDLE |
| 任何有持倉狀態 | 箱體失效 | BOX_INVALID |
| BOX_INVALID | 全部平倉完成 | 重建箱體 → IDLE |
| IDLE | 箱體失效（無持倉） | 重建箱體 → IDLE |

## 十五、掃描優先順序

每次 30 秒掃描，對每個標的依照以下**固定順序**處理：

```
① 同步交易所實際持倉與掛單（交易所是最終真相）
② 檢查停損是否已被交易所執行
③ 檢查第一段出場條件（4H MACD 反轉）
④ 檢查第二段出場條件（1H MACD 反轉，僅 HALF 狀態）
⑤ 檢查加倉條件（僅 LONG / SHORT 狀態且未加倉）
⑥ 檢查開倉條件（僅 IDLE 狀態）
⑦ 更新箱體狀態（上緣延伸 / 失效偵測）
```

**原則：先確認現實 → 先保命 → 再進攻 → 最後更新結構**

同一輪若多個事件同時成立，按此順序處理，**不跳過**、**不並行**。

## 十六、委託執行細節

### 16.1 平倉 50% 數量計算
- 向下取整到交易所允許的最小交易單位
- 寧可少平一點，不要超平

### 16.2 改停損順序（TP1 觸發後）
1. 先確認 TP1 市價平倉已完全成交
2. 撤銷原有停損單
3. 掛出新的保本停損單（停損價 = 平均開倉價）

### 16.3 部分成交處理
- 市價單提交後等待完全成交
- 設定 timeout（建議 30 秒）
- 超時未完全成交 → 取消剩餘部分改掛限價單

## 十七、程式啟動流程（狀態恢復）

```
程式啟動
  │
  ├─ ① 讀取交易所實際持倉與掛單（真相）
  ├─ ② 讀取 state.json（本地記憶）
  ├─ ③ 比對處理：
  │     ├─ 交易所有倉 + 本地有記錄 → 以交易所數量為準，合併本地狀態
  │     ├─ 交易所有倉 + 本地無記錄 → 標記為「孤兒倉位」，只掛保護停損不主動操作
  │     ├─ 交易所無倉 + 本地有記錄 → 清除本地狀態（已被平掉）
  │     └─ 都沒有 → 正常空倉
  ├─ ④ 回寫修正後的 state.json
  └─ ⑤ 載入箱體狀態（從 state.json；若無 state.json 則讀 manual_boxes 當種子）
```

| 判斷 | 運算符 | 說明 |
|------|--------|------|
| 突破上緣（做多） | close **>** ceiling | 嚴格大於，等於不算突破 |
| 跌破下緣（做空） | close **<** floor | 嚴格小於 |

| 回箱重置（空單方向） | close **>** floor | 嚴格大於 |
| MACD 金叉（進場/出場） | hist **>** 0 且 prev_hist **<=** 0 | 當根嚴格正，前根非正 |
| MACD 死叉（進場/出場） | hist **<** 0 且 prev_hist **>=** 0 | 當根嚴格負，前根非負 |
| MACD 多方持續（加倉） | hist **>** 0 | 嚴格大於零 |
| MACD 空方持續（加倉） | hist **<** 0 | 嚴格小於零 |
| 成交量失效條件 | vol **>=** avg_20d × 1.5 | 大於等於 |
| 加倉 4H 確認（多） | low **>** base_ceiling | 嚴格大於，連續 3 根 |
| 加倉 4H 確認（空） | high **<** floor | 嚴格小於，連續 3 根 |
| 上破追蹤進入 | close **>** top_ceiling | 嚴格大於 |
| 上破追蹤結束（假突破） | close **<** top_ceiling | 嚴格小於 |
| 已收線判定 | 當前時間 **>=** K 棒收盤時間 | 未收完的不看 |

## 二十、設計意圖聲明

### 非對稱設計
本策略為**刻意的非對稱設計**：
- 多單側有多層上緣，提供多次進場機會
- 空單側僅固定下緣，只抓大破位

此為策略核心意圖，反映加密市場長期偏多特性。做空本來就該謹慎，只抓確定性高的結構性崩盤。

### 當前版本定位
v1.3 聚焦於**工程穩定性**，所有改動都是修補漏洞，策略邏輯本身（箱體怎麼畫、MACD 怎麼判、什麼時候進出場）與 v1.2 完全一致。

## 二十一、這版已經寫死的重點

1. 所有判斷只看已收線
2. 突破和 MACD 交叉必須出現在同一根 1 小時 K 棒
3. 平均成交量固定看最近 20 根已收日線
4. 自動建箱固定看最近 120 根已收日線（手動箱體僅冷啟動時讀取）
5. 形成期固定 5 根日線（錨點 + 後 4 根）
6. 固定下緣 = 形成期最低價（非單根錨點最低價）
7. 同方向不能一直重複追單（必須先回箱體內再重新突破）
8. 加倉只限一次，且只能在第一段出場前
9. 箱體失效時，不在持倉途中硬切換新箱體
10. 做空止損凍結在進場時的最高上緣，不隨後續延伸變動
11. 所有比較符號已明確定義（見第十九節）
12. 掃描優先順序已固定（見第十五節）
13. 程式重開時以交易所實際持倉為最終真相
14. manual_boxes 為啟動種子模式，非永久覆寫
